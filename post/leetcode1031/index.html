Leetcode.1031 Maximum Sum of Two Non-Overlapping Subarrays

 Posted on September 30, 2020  |  3 minutes  |  449 words  |  Jiayu Carl He

This description/problem statement could be found here. We can take an example in the following to help us to derive the algorithm.

Input: A = [3,8,1,3,2,1,8,9,0], L = 3, M = 2
Output: 29
Explanation: One choice of subarrays is [3,8,1] with length 3, and [8,9] with length 2.

** Brute Froce Solution It should very easy and intuitive to think about this solution. In terms of question like string/array problem, I would recommend everyone to start to think from brute force solution first. Usually, we can do many optimization by using memorization. And we don’t have to write the acutual code, peusdo code should be enough for interview purpose.

  1. we fix an subarray(X) of length L
  2. we look for all other non-overlapping subarrays(Y) of Length M that occur before and after B.

The pesudo code will be like:

result = -1
for subarray X in array A:
	for subarray Y before X:
		result = max(result, sum(X) + sum(Y))
	for subarray Y after X:
		result = max(result, sum(X) + sum(Y))
return result

THis is common brute force approach when we look for pair. This approach will very inefficient. In particular, finding subarray Y for every subarray X is very costly in this case. We can think about how can we do this approach in just o(1).

** Optimization

Naturally, we can think about using memorization to help us to do faster lookup. Let us make some assumption that may help us to do fast look up. In fact, there are many leetcode array questions has similar approach, …….

For each subarray X, if we somehow know the the maximum of the sum(Y) occur before and after X, which I can denote as y1 and y2, then we can just do the following code

for subarray X in array A:
	result = max(result, sum(X) + y1)
	result = max(result, sum(X) + y2)
return result

Now, it comes to how we can do some pre-processing to get all y1 and y2. Let us compute the sum of all possible Y in the A first, which can be denoted as sumY Then we can use two array maxPrefixY and maxSuffixY to store y1 and y2 respectively. To be more details, both array will be len(A). For example, y1 in maxPrefixY with its index 3 means that the maximum of the sum(Y) strictly before index 3 is y1.

In this example, sumY = [(3+8)11, (8+1)9, (1+3)4 (3+2)5 (2+1)3 (1+9)9 (8+9)17 (9+0)9] , and maxPrefixY = [-1,-1,11,11,11,11] , the reason why for the first two value is -1 is because we need at least 2 element to form Y. And maxSuffixY = [17,17,17,17,17,9,-1,-1].
